## 实施阶段

### 阶段 1：基础工具实现（Day 5 内容）

**目标**：Agent 能真正读写文件、执行 shell 命令，端到端跑通"创建 hello.py 并运行"。

#### 1.1 实现 `tools/fs.py` — read / write
- `_read(path, max_bytes=100_000)`:
  - 用 `pathlib.Path` 读文件
  - 超出 max_bytes 时截断并提示 `[已截断，仅显示前 N 字节]`
  - 每行前加行号（`{i:4d}| {line}`），便于后续 edit 定位
  - `FileNotFoundError` → 返回错误字符串而非崩溃
- `_write(path, content)`:
  - 覆盖写入文件
  - 自动创建父目录（`os.makedirs(parent, exist_ok=True)`）
  - 返回确认消息（路径 + 字节数）

#### 1.2 实现 `tools/shell.py` — bash
- `_bash(command, timeout=30)`:
  - `subprocess.run(..., shell=True, capture_output=True, text=True, timeout=timeout)`
  - 返回格式化输出：stdout + stderr + returncode
  - `TimeoutExpired` → 返回超时提示
  - 工作目录为当前进程 cwd（后续 Day10 加沙箱）

#### 1.3 在 `tools/base.py` 注册基础工具
- 取消注释 Day5 的注册代码块，import `read_tool/write_tool/bash_tool`

#### 1.4 验证
```bash
python -m agent.cli --selfcheck          # 应显示 3+14=17 个工具
python -m agent.cli "创建 hello.py 并运行"  # 真后端端到端测试
```

### 阶段 2：更多工具 + v1 里程碑（Day 6 内容）

**目标**：Agent 具备完整的代码操作能力，达到 v1（端到端可用）。

#### 2.1 实现 `tools/more_tools.py` — edit / grep / glob
- `_edit(path, old, new)`:
  - 精确字符串替换：读文件 → 找到 old 字符串 → 替换为 new
  - old 不存在或出现多次 → 返回错误说明
- `_grep(pattern, path=".")`:
  - 调用系统 `rg`（ripgrep），返回匹配行（带文件名+行号）
  - rg 不存在时 fallback 到 `subprocess.run(["grep", "-rn", ...])`
  - 结果超过 50 行时截断
- `_glob(pattern)`:
  - `pathlib.Path().glob()` 查找匹配文件
  - 返回相对路径列表

#### 2.2 注册 Day6 工具
- 在 `build_default_registry()` 中注册 edit_tool/grep_tool/glob_tool

#### 2.3 验证
```bash
# 端到端测试完整工具集
python -m agent.cli "扫描项目所有 Python 文件的 TODO 注释，生成 report.md"
```

### 阶段 3：上下文管理 + 错误恢复 + 更多工具（Day 7 内容）

**目标**：Agent 能处理长任务，具备自愈能力。

#### 3.1 实现错误恢复 `agent/loop.py`
- 将工具执行包裹在 `try/except` 中
- 异常文本作为 observation 回喂模型（让模型自己分析错误并重试）
- 格式：`"工具调用失败：{error_msg}\n请分析错误原因并重试。"`

#### 3.2 实现 `agent/context.py` — maybe_compact
- `estimate_tokens` 已有基础实现（字符数/4），可保留
- `maybe_compact`:
  1. 保留 system（第 0 条）
  2. 如果超出 budget（默认 6000 tokens），将中间早轮 user/assistant/tool 合并
  3. 生成一条 `role="system"` 备忘：`[历史摘要] 任务目标: ... 已完成步骤: ... 关键发现: ...`
  4. 保留最近 K=4 轮原文
  5. 调用后端做摘要（可选：简单截断方案也行）

#### 3.3 在 `agent/loop.py` 集成上下文管理
- 每轮调用 `maybe_compact` 检查 token 预算
- 超预算时替换 messages 为压缩版本后继续

#### 3.4 实现 `tools/more_tools.py` — web_fetch / task_list
- `_web_fetch(url, max_tokens=2000)`:
  - `httpx.get(url)` → `markdownify.markdownify(html)` → 截断到 max_tokens
  - 处理请求异常（超时、404 等）
- `_task_list(action, items=None)`:
  - 维护模块级 `_tasks: list[dict]` 状态
  - action: `add` / `update` / `complete` / `list`
  - 返回格式化的任务清单

#### 3.5 注册 Day7 工具 + 完善 eval
- 注册 web_fetch_tool/task_list_tool
- 实现 `eval/metrics.py` 的核心逻辑
- 补充 `eval/tasks.py` 测试用例

#### 3.6 验证
```bash
# 长任务测试：查外部文档 + 写分析报告
python -m agent.cli "去 requests 官方文档查 retry 机制，写一篇 500 字的笔记保存到 docs/requests-retry.md"
```

### 阶段 4：MCP 协议集成（Day 8 内容）

**目标**：Agent 能接入外部 MCP Server，工具集可插拔扩展。

#### 4.1 实现 `mcp/client.py` — MCPClient
- `start()`: 启动子进程（`subprocess.Popen`），stdin/stdout 接管，发送 `initialize` 请求，等待响应
- `_rpc(method, params)`: 构造 JSON-RPC 2.0 请求 → 发送 → 读取一行响应 → 解析返回
- `list_tools()`: 调用 `tools/list`，返回工具描述列表
- `call_tool(name, arguments)`: 调用 `tools/call`，返回 `result.content[0].text`

#### 4.2 端到端 MCP 验证
- 启动 `mcp/echo_server.py` 作为 MCP server
- 客户端连接 → list_tools → call_tool("echo", {"text": "hello"})
- 验证 `mcp__echo` 工具注册到 ToolRegistry

#### 4.3 集成到 CLI
- 在 `agent/cli.py` 添加 `--mcp-server` 参数，指定外部 MCP server 命令
- cli 启动时自动连接 + 注册 MCP 工具

#### 4.4 验证
```bash
python -m mcp.echo_server | python -m agent.cli --mcp-server "python -m mcp.echo_server" "调用 echo 工具说 hello"
```

### 阶段 5：Skills 技能系统（Day 9 内容）

**目标**：Agent 能按需加载领域知识包（SKILL.md），提升专业领域任务成功率。

#### 5.1 实现 `skills/loader.py` — parse_skill_md + catalog
- `parse_skill_md(text, path)`:
  - 解析 YAML frontmatter（`---` 包围的块）
  - 提取 `name` / `description`
  - body = frontmatter 之后的内容
  - YAML 解析可用 `yaml` 库或手动实现粗略解析
- `skills_catalog(skills)`:
  - 生成 `- {name}: {description}` 格式文本
  - 注入到系统提示词中

#### 5.2 Skills 按需加载机制
- 在 `agent/prompts.py` 中预留 `{SKILLS_CATALOG}` 占位符
- CLI 启动时 `load_skills()` → 生成 catalog → 替补到 SYSTEM_PROMPT
- 模型在 SYSTEM_PROMPT 中看到 skill 清单，可在对话中引用

#### 5.3 验证
```bash
# code-planner SKILL.md 已有完整内容，验证能被正确加载
python -m agent.cli --selfcheck  # 检查 skills 数量
python -c "from skills.loader import load_skills, skills_catalog; print(skills_catalog(load_skills()))"
```

### 阶段 6：安全与最终验收（Day 10 内容）

**目标**：Agent 安全可控，Demo 就绪。

#### 6.1 权限层
- 在 `tools/fs.py` 和 `tools/shell.py` 添加工作目录边界检查
- 写入/执行前检查路径是否在项目目录内
- 危险命令（`rm -rf /`、`sudo` 等）黑名单拦截
- 可选择实现：`--safe-mode` 标志，所有写操作需用户确认

#### 6.2 沙箱执行（可选）
- bash 工具可接入 bwrap/firejail 做轻量沙箱
- 最低限度：timeout + 工作目录限制

#### 6.3 消融实验（评估体系）
- 完善 `eval/tasks.py` 的 E2E_TASKS（补充领域任务）
- 用 `eval/metrics.py` 量化：
  - 工具选择正确率
  - 任务完成率
  - 平均 Token 消耗
  - 平均步数

#### 6.4 最终验证
```bash
# 综合 Demo：分析一个开源仓库
python -m agent.cli "分析 https://github.com/xxx/yyy 仓库结构，生成架构图、代码质量报告"
```
