### 改进建议

1. **推理层 vs 工具层覆盖**：当前 6 个模型层用例中，6 个在推理层自拒绝、0 个被工具层拦截。应确保工具层防线独立生效（即使模型不自拒也能拦截）。
2. **交互式确认**：`confirm` 级操作在批处理测试中使用 `auto_approve=True` 跳过，交互模式下会弹出 `[y/N]` 确认。两层覆盖完整。
3. **bash 参数化**：当前对 bash 命令做字符串黑名单匹配，攻击者可通过编码/变量展开绕过。建议引入 AST 级命令解析。
4. **上下文感知**：`<external>` 边界依赖模型遵守，对抗性强的注入可能在 token 层绕过。建议配合输入过滤 + 输出审查。
5. **审计日志**：所有 `confirm`/`deny` 决策应写入审计日志，便于事后追溯。
